## 2026-05-26 - React Component Re-rendering in Grid.tsx
**Learning:** In a highly interactive grid with thousands of cells, small inline objects or functions can severely impact performance. In `components/Grid.tsx`, `EnhancedCell` is wrapped in `React.memo()`, but passing an inline empty object (`{}`) for default styles or an inline function for `onFillStart` was breaking the shallow comparison, causing every visible cell to re-render whenever the grid re-rendered (e.g. from state changes).
**Action:** Always extract static default values (like an empty object `EMPTY_STYLE`) outside the component and use `useCallback` for event handlers passed to many children to ensure referential stability and keep React rendering fast.

## 2024-05-28 - React.memo Optimization\n**Learning:** Wrapping complex UI elements like charts (in Visualization.tsx) and their parent containers (Dashboard.tsx) in `React.memo` effectively limits their re-renders. This is particularly useful when these components receive complex objects as props (like configurations or entire sheet data). To maximize this benefit, parent components must provide referentially stable props.\n**Action:** Add `React.memo` to high-cost rendering components. Ensure parent components pass stable references for callbacks and objects using `useCallback` and `useMemo`.
## 2024-05-20 - [Fix EnhancedCell Memoization Breakdown during Selection]
**Learning:** React.memo on cell components fails during dragging because a callback (`handleFillStart`) passed to the cell includes `selectedRange` in its dependency array. Since `selectedRange` updates rapidly during mouse drag, the callback's reference changes, triggering O(N*M) cell re-renders and causing severe UI jank.
**Action:** Use a mutable ref (`useRef` + `useLayoutEffect`) to track rapidly changing state like `selectedRange` inside stable callbacks to preserve memoization of child components.
## 2026-05-31 - Referential Stability for Heavy Child Components\n**Learning:** High-level wrapper components (like `App.tsx`) often pass utility functions (like `addToDashboard` or `removeFromDashboard`) to many heavy, frequently-rendering children. If these utilities aren't wrapped in `useCallback`, every state change in the parent (e.g., cell edit, selection change) recreates the utility function, invalidating the `React.memo` of child components and causing deep, expensive re-renders.\n**Action:** Always wrap utility functions passed to heavy child components in `useCallback` and verify dependency arrays to ensure referential stability.
## 2026-06-01 - Prevent Excessive Grid Re-renders
**Learning:** High-frequency event tracking (like tracking `hoverCell` on every mouse move over a cell) inside a heavy parent component (`Grid`) causes severe O(N*M) re-renders, even if child components (`EnhancedCell`) are memoized. Additionally, failing to memoize the data object or callback functions passed to the `Grid` from `App.tsx` invalidates `React.memo` completely.
**Action:** Avoid tracking `hoverCell` in React state when purely visual hover effects can be handled by CSS. Always wrap the `Grid` component in `React.memo()` and verify that all complex objects (like `gridData`) and callback functions passed from `App.tsx` to `Grid` are referentially stable using `useMemo` and `useCallback` with correct dependency arrays.
## 2026-06-03 - [Fix EnhancedCell Memoization Breakdown during Edits]
**Learning:** `React.memo` on child components fails if callback props (like `onCellEdit`) passed from the parent (`Grid`) are unstable. Because `onCellEdit` in `App.tsx` has dependencies like `workbook` and `currentSheetData` which change frequently, its reference constantly changes, breaking memoization for all visible `EnhancedCell`s during every edit.
**Action:** In the intermediary component (`Grid`), use a mutable ref (`useRef` + `useLayoutEffect`) to track the latest unstable callback, and define a referentially stable wrapper callback (`useCallback` with an empty dependency array) to pass down to the memoized children.
